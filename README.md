# Peel — Food Waste Solver

Hedera Agentic Society Hackathon · 11–12 April 2026.

Two parallel workstreams, one shared token primitive:

| Branch | PRD | Status |
|---|---|---|
| `market` | [`PRD-2-Market.md`](./PRD-2-Market.md) | **Primary build.** Agent-to-agent inventory economy. |
| `programme` | [`PRD-1-Programme.md`](./PRD-1-Programme.md) | Background stub. Performance-credit ranking. |

Both branches share `shared/` — the Hedera client factory, the four `RAW_*` token registry, and the HCS topic registry. Everything else is isolated per branch so the two worktrees never collide.

## Layout

```
shared/           cross-workstream: hedera client, tokens, topics, policies, types
market/           PRD-2 build — Kitchen Trader agents, app.html, bootstrap scripts
programme/        PRD-1 stub — Kitchen/Regulator agents, recipes, period-close runner
index.html        public landing page (Peel brand)
tasks/todo.md     session-level task tracking (per CLAUDE.md convention)
```

## Running two sessions in parallel

From this folder:

```bash
# One-time: create sibling worktrees for each workstream
git worktree add ../peel-market market
git worktree add ../peel-programme programme

# Terminal 1 — Market (primary build)
cd ../peel-market
cp ../aaFood\ Waste\ Solver/.env .
npm install
# launch Claude here → PRD-2-Market.md is the brief

# Terminal 2 — Programme (background stub)
cd ../peel-programme
cp ../aaFood\ Waste\ Solver/.env .
npm install
# launch Claude here → PRD-1-Programme.md is the brief
```

Each session operates on its own branch, writes only inside its own subfolder, and merges back to `main` when done. `shared/` is stable contract — change it only if both branches agree.

## Canonical tokens

Four `RAW_*` fungible tokens, shared between both PRDs:

- `RAW_RICE`
- `RAW_PASTA`
- `RAW_FLOUR`
- `RAW_OIL`

IDs are generated by `market/scripts/bootstrap-tokens.ts` on first run and written into `shared/hedera/generated-tokens.json` (gitignored — each developer bootstraps their own testnet tokens).
